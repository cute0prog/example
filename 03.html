1. 변수

I. 변수의 개념과 생성

변수(Variable)는 프로그램 내에서 특정 값을 저장하는 ’보관 상자’와 같습니다. R에서는 <- 연산자를 사용하여 변수에 값을 할당(저장)합니다.

# 'a'라는 이름의 변수(보관 상자)에 숫자 10을 저장합니다.
a <- 10

II. 변수의 내용 확인하기

# [코드 3-1] 변수 내용 확인 실습
total <- 5050

# 방법 1: 변수 이름 직접 실행 (가장 간단)
total
[1] 5050
# 방법 2: print() 함수 사용
print(total)
[1] 5050
# 방법 3: cat() 함수로 다른 텍스트와 함께 출력
cat('합계 :', total)
합계 : 5050

III. 변수를 이용한 산술연산

변수는 저장된 값을 대표하므로, 변수끼리 연산하면 변수에 저장된 값들이 연산됩니다.

# [코드 3-2] 변수 간 산술연산
a <- 10
b <- 20
c <- a + b  # a의 값(10)과 b의 값(20)을 더해 c에 저장합니다.

print(a)
[1] 10
print(b)
[1] 20
print(c)
[1] 30

IV. 변수에 저장될 수 있는 값의 종류 (자료형)

변수에는 숫자, 문자, 논리값 등 다양한 종류의 값을 저장할 수 있습니다.

숫자형과 문자형

  • 숫자형(Numeric): 산술연산이 가능한 숫자입니다.

  • 문자형(Character): 따옴표(’ 또는 “)로 감싸진 텍스트이며, 산술연산이 불가능합니다.

# [코드 3-4] 숫자형과 문자형 연산 비교
age.1 <- 20         # 숫자 20 저장
age.2 <- 25         # 숫자 25 저장
print(age.1 + age.2) # 숫자끼리의 덧셈은 정상 실행됩니다.
[1] 45
name.1 <- 'John'    # 문자 'John' 저장
print(name.1)
[1] "John"
grade.1 <- '3'      # 모양은 숫자지만 따옴표로 감쌌으므로 문자 '3' 입니다.
# print(age.1 + grade.1) # 숫자와 문자 간 덧셈은 에러를 발생시킵니다. 주석을 풀고 실행해 보세요.

논리형

  • 논리형(Logical): TRUE (참) 또는 FALSE (거짓) 값을 가집니다. 보통 비교 연산의 결과로 나타납니다.

  • 산술연산에 사용될 경우 TRUE는 1, FALSE는 0으로 취급됩니다.

# [코드 3-5] 논리형 값 실습
print(5 > 3)       # 5가 3보다 크므로 TRUE
[1] TRUE
print(2 > 7)       # 2가 7보다 크지 않으므로 FALSE
[1] FALSE
print(TRUE + TRUE) # 1 + 1과 같으므로 2가 출력됩니다.
[1] 2
a <- TRUE
b <- F  # FALSE는 F로 축약해서 쓸 수 있습니다.
print(a + b) # 1 + 0과 같으므로 1이 출력됩니다.
[1] 1

LAB: 소금물의 농도 구하기

변수와 cat() 함수를 활용하여 소금물의 농도를 계산하고 출력해봅시다.

소금물 농도(%) = (소금의 양 / (소금의 양 + 물의 양)) * 100

# 1. 소금 50g, 물 100g일 때
salt <- 50
water <- 100
result <- salt / (salt + water) * 100
cat("소금 =", salt, ", 물 =", water, ": 농도 =", result, "%")
소금 = 50 , 물 = 100 : 농도 = 33.33333 %
# 2. 소금 70g, 물 110g일 때 (변수 값을 바꾸어 재실행)
salt <- 70
water <- 110
result <- salt / (salt + water) * 100
cat("\n소금 =", salt, ", 물 =", water, ": 농도 =", result, "%")

소금 = 70 , 물 = 110 : 농도 = 38.88889 %

2. 벡터

I. 벡터의 개념과 생성

벡터(Vector)는 동일한 자료형의 값 여러 개를 한 번에 저장하는 기능입니다. c() 함수를 사용하여 만듭니다.

# [코드 3-10] 10명 학생의 성적을 벡터로 관리하기
score <- c(68, 95, 83, 76, 90, 80, 85, 91, 82, 70)

# 벡터를 사용하면 sum(), mean() 같은 함수로 쉽게 계산 가능
total <- sum(score)
avg <- mean(score)

cat("총점:", total, "\n")
총점: 820 
cat("평균:", avg)
평균: 82

하나의 벡터에는 동일한 자료형만 저장됩니다. 만약 숫자와 문자를 섞으면 모든 값이 문자로 변환됩니다.

# 숫자와 문자를 섞어 벡터 생성
w <- c(1, 2, 3, 'a', 'b', 'c')
w # 모든 요소가 문자형 "1", "2", "3"으로 변환된 것을 확인하세요.
[1] "1" "2" "3" "a" "b" "c"

II. 특별한 벡터 만들기

  • 연속적인 숫자 벡터: 콜론(:) 사용
  • 일정한 간격의 숫자 벡터: seq() 함수 사용
  • 반복된 숫자 벡터: rep() 함수 사용
v1 <- 50:90        # 50부터 90까지의 정수 벡터
v3 <- seq(1, 10, 3)  # 1에서 시작해 3씩 증가하여 10을 넘지 않는 벡터
v4 <- seq(0.1, 1.0, 0.1) # 0.1부터 1.0까지 0.1 간격의 벡터
v5 <- rep(1, times = 5)  # 1을 5번 반복
v6 <- rep(1:3, times = 3) # 1,2,3 벡터를 3번 반복
v7 <- rep(c(1,5,9), each = 3) # 1,5,9 각 요소를 3번씩 반복

v7
[1] 1 1 1 5 5 5 9 9 9

III. 벡터 원소 다루기 (인덱싱)

벡터에 저장된 각각의 값을 원소(element)라고 하며, 인덱스(index)를 통해 접근할 수 있습니다. R의 인덱스는 1부터 시작합니다.

# [코드 3-13, 3-14] 벡터 인덱싱 실습
d <- c(1, 4, 3, 7, 8)

d[1]        # 첫 번째 원소
[1] 1
d[3]        # 세 번째 원소
[1] 3
d[c(1, 3, 5)] # 첫 번째, 세 번째, 다섯 번째 원소
[1] 1 3 8
d[2:4]      # 두 번째부터 네 번째까지의 원소
[1] 4 3 7
d[-2]       # 두 번째 원소를 '제외'한 나머지
[1] 1 3 7 8

IV. 벡터 원소에 이름 붙이기

names() 함수를 사용하여 벡터의 각 원소에 이름을 붙일 수 있습니다. 이름으로도 원소에 접근이 가능해져 코드가 더 직관적으로 변합니다.

# [코드 3-15] 이름으로 값 추출하기
sales <- c(640, 720, 680, 540) # 1~4월 매출액
names(sales) <- c('M1', 'M2', 'M3', 'M4')

sales          # 이름과 함께 출력
 M1  M2  M3  M4 
640 720 680 540 
sales['M2']    # 이름으로 2월 매출액 추출
 M2 
720 
sales[c('M1','M4')] # 여러 월의 매출액 추출
 M1  M4 
640 540

LAB: 적금 만기 금액 계산하기

벡터와 이름붙이기를 활용하여 여러 사람의 적금 만기 금액을 계산해봅시다. 만기 금액 = 원금 * (1 + 연이율/100) ^ 기간

# 1. 이름, 원금, 이율, 기간을 각각의 벡터로 만들기
customer <- c('kim', 'lee', 'park', 'choi', 'seo')
deposit <- c(5000000, 4500000, 4000000, 5500000, 6000000)
rate <- c(3.5, 3, 4, 5, 4.5)
period <- c(2, 2, 5, 7, 4)

# 2. 벡터에 고객 이름 설정
names(deposit) <- customer
names(rate) <- customer
names(period) <- customer

# 3. 특정 고객('kim')의 만기 금액 계산
who <- 'kim'
kim_sum <- deposit[who] * (1 + rate[who] / 100) ^ period[who]
kim_sum
    kim 
5356125 
# 4. 모든 고객의 만기 금액 한 번에 계산
all_sum <- deposit * (1 + rate / 100) ^ period
all_sum
    kim     lee    park    choi     seo 
5356125 4774050 4866612 7739052 7155112

3. 함수

I. 함수의 개념과 매개변수

함수(Function)는 특정 기능을 수행하는 코드의 묶음입니다. 입력값(인수, Argument)을 받아 정해진 계산을 수행한 후 결과값을 돌려줍니다. 함수에 입력값을 전달받는 변수를 매개변수(Parameter)라고 합니다.

sort() 함수로 매개변수 사용법을 알아봅시다.

# [코드 3-18] sort() 함수 사용법
d <- c(1, 7, 4, 2, 3)

# 1. 기본 사용 (오름차순 정렬)
sort(d)
[1] 1 2 3 4 7
# 2. 매개변수 'decreasing'을 TRUE로 설정하여 내림차순 정렬
sort(d, decreasing = TRUE)
[1] 7 4 3 2 1
# sort() 함수의 정식 문법은 sort(x, decreasing) 입니다.
# x: 정렬할 벡터 (필수 매개변수)
# decreasing: 내림차순 여부 (선택적 매개변수, 기본값: FALSE)
# 매개변수 이름을 생략하고 순서대로 값을 입력할 수 있으며, 기본값이 있는 선택적 매개변수는 생략 가능합니다.

LAB: paste() 함수로 문자열 만들기

paste() 함수는 여러 텍스트나 변수를 하나로 합쳐주는 유용한 함수입니다.

# 1. 기본적인 사용법 ('sep' 매개변수로 구분자 지정)
str <- paste('good', 'morning', sep = ' / ')
str
[1] "good / morning"
# 2. 변수들 연결하기
a <- '나의 나이는'
b <- 20
c <- '입니다'
paste(a, b, c, sep = ' ') # sep의 기본값은 공백(' ')입니다.
[1] "나의 나이는 20 입니다"
# 3. 벡터와 값 연결하기
a <- 1:12
b <- '월'
months <- paste(a, b, sep = '') # 벡터 a의 모든 원소에 '월'을 붙여 새 벡터 생성
months
 [1] "1월"  "2월"  "3월"  "4월"  "5월"  "6월"  "7월"  "8월"  "9월"  "10월"
[11] "11월" "12월"

실전분석: 월별 매출액 분석하기

지금까지 배운 변수, 벡터, 함수, 인덱싱을 모두 활용하여 월별 매출액 데이터를 분석해봅시다.

# 1. 1년치 월별 매출액 데이터를 벡터 'sales'에 저장
sales <- c(750, 740, 760, 680, 700, 710, 850, 890, 700, 720, 690, 730)

# paste() 함수를 이용해 1월~12월 이름을 만들어 sales 벡터에 적용
names(sales) <- paste(1:12, '월', sep = "")
sales

 750  740  760  680  700  710  850  890  700  720  690  730 
# 2. 7월의 매출액 알아보기
sales['7월']
7월 
850 
# 3. 1월과 2월의 매출액 합계 알아보기
sales['1월'] + sales['2월']
 1월 
1490 
# 4. 매출액이 가장 많은 달과 그 금액 알아보기
# sort()로 내림차순 정렬 후 첫 번째 값을 확인
max_month_sales <- sort(sales, decreasing = TRUE)
max_month_sales[1]
8월 
890 
# 5. sum()과 인덱싱으로 상반기(1~6월) 매출액 합계 알아보기
sum(sales[1:6])
[1] 4340
